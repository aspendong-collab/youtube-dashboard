# 导航栏结构优化方案

## 📋 当前结构分析

### 现有导航结构（过于分散）

```
📊 仪表盘
├── 📹 视频管理
├── 📊 整体看板
├── 📹 单个视频
└── 🔥 爆款提醒

📈 数据分析
├── 🎯 SEO 优化
├── ⏱️ 时长分析
├── 🕐 发布时间
└── 🏷️ 标签分析

💬 深度分析
├── 😊 情感分析
├── 👥 用户画像
└── 🔍 评论分析

⚙️ 设置
├── 🔑 API 配置
└── 📊 数据源管理
```

### 问题分析

1. **分类逻辑不清晰**
   - "视频管理"、"单个视频"、"爆款提醒" 都是视频相关的，但分散在不同分组
   - "数据分析" 和 "深度分析" 的界限模糊
   - SEO、时长、发布时间、标签都是内容优化相关，但分散

2. **父子维度分离**
   - "整体看板" 和 "单个视频" 应该是父子关系
   - SEO、时长、发布时间、标签都是内容维度的分析
   - 情感分析、用户画像、评论分析都是用户维度的分析

3. **信息过于分散**
   - 4 个分组，13 个页面
   - 用户需要在多个分组间跳转才能完成相关任务
   - 不符合用户的工作流程

## 🎯 新结构设计（整合版）

### 方案 A: 按工作流程分组

```
🏠 概览与管理
├── 📊 整体数据看板
└── 📹 视频管理
    └── 📹 单个视频详情
    └── 🔥 爆款提醒

📈 内容表现分析
├── 🎯 SEO 优化分析
├── ⏱️ 时长分析
├── 🕐 发布时间分析
└── 🏷️ 标签分析

👥 用户与互动分析
├── 🔍 评论深度分析
├── 😊 情感分析
└── 👥 用户画像

⚙️ 系统设置
├── 🔑 API 配置
└── 📊 数据源管理
```

**优点**:
- 按工作流程分组（管理 → 内容分析 → 用户分析 → 设置）
- 视频相关功能整合在一起
- 内容分析和用户分析界限清晰

**缺点**:
- 子页面层级不够明显

## 🏆 推荐方案：方案 B（按分析维度分组）

### 最终导航结构

```
📊 数据概览
├── 📊 整体数据看板
└── 📹 视频管理
    ├── 📹 单个视频详情
    └── 🔥 爆款提醒

📝 内容优化
├── 🎯 SEO 优化分析
├── ⏱️ 时长分析
├── 🕐 发布时间分析
└── 🏷️ 标签分析

💬 用户互动
├── 🔍 评论深度分析
├── 😊 情感分析
└── 👥 用户画像

⚙️ 系统设置
├── 🔑 API 配置
└── 📊 数据源管理
```

### 分组逻辑说明

#### 📊 数据概览
- **核心功能**: 查看整体数据和管理视频
- **包含页面**:
  - 整体数据看板：查看所有视频的总体数据
  - 视频管理：添加、删除、查看视频列表
  - 单个视频详情：查看单个视频的详细数据
  - 爆款提醒：查看数据突增的视频

#### 📝 内容优化
- **核心功能**: 分析和优化视频内容
- **包含页面**:
  - SEO 优化分析：标题、描述、标签的 SEO 效果
  - 时长分析：视频时长与表现的关系
  - 发布时间分析：最佳发布时间
  - 标签分析：标签效果分析

#### 💬 用户互动
- **核心功能**: 分析用户行为和互动
- **包含页面**:
  - 评论深度分析：评论内容、趋势、热点
  - 情感分析：评论的情感倾向
  - 用户画像：活跃用户、用户特征

#### ⚙️ 系统设置
- **核心功能**: 系统配置和数据管理
- **包含页面**:
  - API 配置：YouTube API 密钥配置
  - 数据源管理：数据源管理

## 🎯 实施建议

### 1. 导航栏实现方式

#### 方案 1: 使用 expander 实现子菜单
```python
with st.sidebar.expander("📊 数据概览"):
    if st.button("📊 整体数据看板"):
        st.session_state.current_page = "overall_dashboard"
    if st.button("📹 视频管理"):
        st.session_state.current_page = "video_management"
    with st.sidebar.expander("└── 视频相关"):
        if st.button("📹 单个视频详情"):
            st.session_state.current_page = "video_detail"
        if st.button("🔥 爆款提醒"):
            st.session_state.current_page = "alerts"
```

#### 方案 2: 使用 selectbox 实现主导航 + 页面内子导航
```python
# 侧边栏：主导航
page = st.sidebar.selectbox(
    "选择功能",
    ["📊 数据概览", "📝 内容优化", "💬 用户互动", "⚙️ 系统设置"]
)

# 页面内：子导航
if page == "📊 数据概览":
    sub_page = st.selectbox(
        "选择页面",
        ["📊 整体数据看板", "📹 视频管理", "📹 单个视频详情", "🔥 爆款提醒"]
    )
```

#### 方案 3: 使用多级按钮（推荐）
```python
# 主分类按钮
if st.sidebar.button("📊 数据概览"):
    st.session_state.category = "overview"
    st.rerun()

# 根据分类显示子页面
if st.session_state.category == "overview":
    st.sidebar.markdown("---")
    st.sidebar.write("#### 子页面")
    if st.sidebar.button("📊 整体数据看板"):
        st.session_state.current_page = "overall_dashboard"
```
